# Life OS Designer — Expert Skill

## The Life OS Model

A Life Operating System is a single coherent framework that:
1. **Captures** everything competing for your attention
2. **Processes** it into the right bucket at the right time
3. **Executes** with clear prioritization across all domains
4. **Reviews** what's working and adjusts continuously

## The Weekly Operating Rhythm

```
MONDAY — Orientation (60 min)
├── Review last week's scorecard (completed? incomplete? why?)
├── Set this week's Top 3 per domain (18 items max)
├── Block calendar for deep work sessions
└── Clear inbox (email + Obsidian + physical)

TUESDAY–THURSDAY — Execution
├── Morning: Most important single task (no meetings until noon if possible)
├── Afternoon: Meetings, communication, collaboration
└── Evening: Capture tomorrow's top 3

FRIDAY — Ship + Reflect (90 min)
├── Deliver anything with Friday deadlines
├── Weekly Review template (Obsidian)
├── Update consulting + business trackers
└── Prepare for weekend intentionality

WEEKEND — Recharge + Faith + Personal
├── No work email Saturday (enforce with app limits)
├── Faith practice Saturday evening / Sunday morning
├── Family priority
└── 1 hour Sunday: preview next week
```

## Goal Cascade Architecture

```
10-Year Vision → 3-Year Milestones → Annual OKRs → Quarterly Rocks → Weekly Goals → Daily Top 3

Each level should be reviewable at the level above:
- Annual OKRs checked quarterly (are we on track?)
- Quarterly Rocks checked weekly (what moves the needle this week?)
- Weekly Goals checked daily (what's my single most important task today?)
```

### OKR Format for Life Domains
```
Objective: [Ambitious direction]
KR1: [Measurable result — number by date]
KR2: [Measurable result — number by date]
KR3: [Measurable result — number by date]

Example:
Objective: Establish myself as the go-to AI automation consultant in my market
KR1: Close 2 new consulting clients by June 30
KR2: Publish 1 case study per quarter (4 total by year end)
KR3: 10 qualified discovery calls completed by September 30
```

## Cross-Domain Integration Points

These are the system links that make the OS coherent rather than fragmented:

1. **Consulting → Business** — Skills and templates built for clients become business IP
2. **Work → Consulting** — Corporate experience validates consulting positioning
3. **Health → All** — Energy and cognitive performance is the foundation everything runs on
4. **Faith → All** — Values filter drives which clients/projects to take
5. **Personal → Business** — Financial goals drive business revenue targets

## Quarterly Review Protocol (4hrs, end of each quarter)

```
QUARTER IN REVIEW:
[ ] What did I accomplish in each domain? (vs. OKRs)
[ ] What was I going to do that I didn't? Why?
[ ] What surprised me — good and bad?
[ ] What am I carrying that I should drop?
[ ] What deserves more of my time next quarter?

QUARTER AHEAD:
[ ] Set 3 Quarterly Rocks per domain (max 18 total)
[ ] Block deep work weeks in calendar
[ ] Identify which relationships need investment
[ ] Financial projection: consulting revenue, expenses, savings rate
[ ] One "anti-goal": what am I explicitly NOT doing this quarter?
```

## Anti-Patterns

1. **Domain Isolation** — Running each life area as a separate silo with no integration. Your health affects your business affects your consulting affects your faith.
2. **The Infinite Backlog** — 200 tasks is not a system, it's anxiety in list form. Weekly Top 3 per domain is the limit.
3. **Review Without Adjustment** — Weekly reviews that produce no changes are rituals, not systems. Every review should change at least one thing.
4. **Perfect System Paralysis** — Spending more time designing the system than living it. Good enough + consistent > perfect + abandoned.
